SSH DÉBUTANT
Objectif
Se connecter au serveur OverTheWire via SSH et lire le fichier readme pour obtenir le mot de passe du niveau suivant.
Théorie – SSH

SSH (Secure Shell) est un protocole réseau qui permet de se connecter à un ordinateur distant de manière chiffrée et sécurisée.

Le port standard est le 22, mais ici OverTheWire utilise le port 2220.

Un port, c'est comme un numéro de bureau dans un bâtiment : il indique à l'ordinateur quel service utiliser (SSH, HTTP, FTP…).

Syntaxe : ssh utilisateur@hôte -p port

FICHIER SPÉCIALTIRET
Objectif
Lire le contenu d'un fichier dont le nom est - (un simple tiret), ce qui est piégeux car le shell confond ce nom avec une option de commande.
Exploration

cat ./-
263JGJPfgU6LtdEvgfWU1XP5yac29mFx
Pourquoi ./ ?

Quand on tape cat -, le shell pense que - est une option de commande (comme -v, -n…) et attend une entrée sur stdin.

En préfixant par ./, on force le shell à interpréter ./- comme un chemin relatif vers un fichier.

Le . signifie "le répertoire courant", donc ./fichier = "le fichier dans le dossier où je suis".

BANDIT / NIVEAU 2
ESPACESÉCHAPPEMENT
Objectif
Lire un fichier dont le nom contient des espaces : --spaces in this filename--. Les espaces sont des caractères spéciaux en shell.
Le problème avec les espaces

En shell, les espaces séparent les arguments. Si tu tapes cat --spaces in this filename--, le shell croit que tu lui passes 4 fichiers différents : --spaces, in, this, filename--.

Pour lire un fichier avec des espaces, deux solutions :

1. Échapper chaque espace avec \ : cat ./--spaces\ in\ this\ filename--

2. Entourer le nom de guillemets : cat "./--spaces in this filename--"

ls
--spaces in this filename--

BASE64DÉCODAGE
Objectif
Le fichier data.txt contient des données encodées en Base64. Il faut les décoder pour obtenir le mot de passe.
Qu'est-ce que Base64 ?

Base64 est un encodage (pas un chiffrement !) qui transforme n'importe quelle donnée binaire en une chaîne de caractères alphanumériques.

Une chaîne Base64 se reconnaît souvent par ses = ou == à la fin (padding).

Pour décoder : base64 -d fichier ou echo "chaine" | base64 -d

ROT13TR
Objectif
Le fichier data.txt contient du texte chiffré en ROT13 (rotation de 13 lettres de l'alphabet). Il faut le déchiffrer avec la commande tr.
ROT13

ROT13 est un chiffrement par substitution très simple : chaque lettre est remplacée par la lettre 13 positions plus loin dans l'alphabet.

Exemple : A→N, B→O, N→A… C'est son propre inverse (appliquer ROT13 deux fois = texte d'origine).

Sous Linux, on utilise tr (translate) pour remplacer des caractères :

tr 'A-Za-z' 'N-ZA-Mn-za-m' = décale toutes les lettres de 13 positions.

NETCATTCP PORT 30000
Objectif
Envoyer le mot de passe du niveau 14 au port 30000 sur localhost. Le service répondra avec le mot de passe du niveau 15.
netcat (nc)

netcat (nc) est le "couteau suisse des réseaux". Il permet d'ouvrir des connexions TCP/UDP brutes, d'écouter sur un port, d'envoyer des données…

Ici on s'en sert en mode client : nc localhost 30000 = se connecte au port 30000.

Le pipe | avec echo permet d'envoyer le mot de passe automatiquement sans le taper manuellement :

echo "MOT_DE_PASSE" | nc localhost 30000

Comment ça fonctionne ?
1
echo produit le mot de passeecho "4wcYUJFw0k0..." affiche le texte sur stdout.
2
Le pipe | redirigeLa sortie de echo devient l'entrée de nc (stdin de netcat).
3
nc envoie les donnéesnetcat se connecte au port 30000 et envoie le texte reçu via le pipe.
4
Le serveur répondLe service vérifie le mot de passe et retourne le suivant si correct.
Terminal
bandit14@bandit
echo "4wcYUJFw0k0XLShlDzztnTBHiqxU3b3e" | nc localhost 30000
