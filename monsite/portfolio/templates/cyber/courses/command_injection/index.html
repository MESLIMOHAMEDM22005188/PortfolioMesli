🧨 OS Command Injection

🎯 Définition

Une OS Command Injection survient lorsqu’une application exécute des commandes système en utilisant des entrées utilisateur non filtrées, permettant à un attaquant d’exécuter du code arbitraire sur le serveur.

⚠️ Impact

  • Exécution de commandes arbitraires
  • Compromission complète du serveur
  • Pivot réseau vers d’autres machines

🧠 Pourquoi ça arrive

Cette vulnérabilité est souvent causée par la concaténation directe d’entrées utilisateur dans des fonctions système telles que system(), exec() ou popen().

🔍 Reconnaître une Command Injection

  • Fonctions système exposées
  • Fonctionnalités de diagnostic (ping, traceroute)
  • Manipulation de chemins ou fichiers

🛡️ Prévention

  • Whitelist stricte des entrées
  • APIs sans invocation de shell
  • Séparation des arguments

💡 Détection des Injections de Commandes OS — Mon Apprentissage

Quand on cherche à détecter des vulnérabilités basiques d’injection de commandes système (OS Command Injection), on suit à peu près le même processus que pour les exploiter. On essaie d’ajouter nos propres commandes via différentes méthodes d’injection. Si la sortie de la commande change par rapport au résultat attendu, cela veut dire qu’on a réussi à exploiter la faille.

Pour des injections plus avancées, ce n’est pas toujours aussi simple. On peut alors utiliser des techniques comme le fuzzing ou la revue de code pour repérer des failles potentielles. Ensuite, on construit petit à petit notre payload jusqu’à réussir l’injection. Mais ici, on va se concentrer sur les injections basiques, où on contrôle une entrée utilisateur qui est directement utilisée dans une commande système, sans aucune protection ni nettoyage.

Exemple pratique : L’exercice Host Checker

Dans l’exercice qu’on va voir, on a une application web avec un outil appelé « Host Checker ». Il nous demande une adresse IP pour vérifier si elle est active.

On peut tester avec l’adresse IP locale, 127.0.0.1, et comme prévu, on reçoit le résultat de la commande ping qui nous confirme que la machine locale répond bien.

Même si on n’a pas accès au code source, on peut deviner que l’adresse IP qu’on entre est utilisée dans une commande ping, vu la sortie qu’on obtient. Par exemple, la commande pourrait ressembler à ça : 

ping -c 1 NOTRE_ENTRÉE

Si l’entrée qu’on fournit n’est pas nettoyée ou échappée avant d’être utilisée, on peut essayer d’injecter une autre commande arbitraire. C’est ce qu’on va tester pour voir si l’application est vulnérable à une injection de commande OS.

Comment injecter une commande ?

Pour ajouter une commande supplémentaire à celle prévue, on peut utiliser plusieurs opérateurs d’injection, comme :

Opérateur Caractère Encodage URL Résultat de l’exécution
Point-virgule ; %3b Les deux commandes sont exécutées
Nouvelle ligne \n %0a Les deux commandes sont exécutées
Arrière-plan & %26 Les deux commandes, la seconde sortie souvent affichée en premier
Pipe | %7c Les deux commandes, mais seule la sortie de la seconde est visible
ET logique (AND) && %26%26 Les deux commandes, si la première réussit
OU logique (OR) || %7c%7c La seconde commande, si la première échoue
Sous-shell `` %60%60 Les deux commandes (Linux uniquement)
Sous-shell $() %24%28%29 Les deux commandes (Linux uniquement)

On écrit d’abord notre entrée normale (par exemple une IP), puis on ajoute un de ces opérateurs, et enfin la commande qu’on veut injecter.

Ces opérateurs fonctionnent généralement pour toutes les applications web, peu importe le langage, le framework ou le serveur utilisé. Que ce soit une appli PHP sur Linux, .Net sur Windows, ou NodeJS sur macOS, nos injections devraient marcher.

Attention : Le point-virgule ; ne fonctionne pas toujours sous l’invite de commandes Windows (CMD), mais il marche sous PowerShell.

Dans la suite, on va essayer d’utiliser un de ces opérateurs pour exploiter l’exercice Host Checker et voir si on peut injecter une commande.

🧪 Exploitation — Injection via le séparateur ;

Jusqu’ici, nous avons identifié que l’application Host Checker pouvait être vulnérable à une OS Command Injection. Nous allons maintenant tester une première tentative d’injection en utilisant l’opérateur point-virgule (;), qui permet de chaîner plusieurs commandes système.

Injecter une commande

L’idée est simple : si l’application exécute une commande de type ping avec une entrée utilisateur, nous tentons d’ajouter une seconde commande après un point-virgule. 

127.0.0.1; whoami

La commande finale exécutée côté serveur deviendrait alors : 

ping -c 1 127.0.0.1; whoami

Validation locale

Avant toute exploitation web, il est crucial de comprendre le comportement de la commande au niveau système. Sur une machine Linux : 

ping -c 1 127.0.0.1; whoami
PING 127.0.0.1 (127.0.0.1)
64 bytes from 127.0.0.1: icmp_seq=1 ttl=64 time=1.03 ms

--- ping statistics ---
1 packets transmitted, 1 received, 0% packet loss

21y4d

✅ Les deux commandes s’exécutent correctement, et la sortie de whoami est bien affichée.
