🧨 OS Command Injection

🎯 Définition

Une OS Command Injection survient lorsqu’une application exécute des commandes système en utilisant des entrées utilisateur non filtrées, permettant à un attaquant d’exécuter du code arbitraire sur le serveur.

⚠️ Impact

  • Exécution de commandes arbitraires
  • Compromission complète du serveur
  • Pivot réseau vers d’autres machines

🧠 Pourquoi ça arrive

Cette vulnérabilité est souvent causée par la concaténation directe d’entrées utilisateur dans des fonctions système telles que system(), exec() ou popen().

🔍 Reconnaître une Command Injection

  • Fonctions système exposées
  • Fonctionnalités de diagnostic (ping, traceroute)
  • Manipulation de chemins ou fichiers

🛡️ Prévention

  • Whitelist stricte des entrées
  • APIs sans invocation de shell
  • Séparation des arguments
